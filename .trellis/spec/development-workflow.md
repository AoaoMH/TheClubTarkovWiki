# Development Workflow Practices

跨层开发实践约束，适用于所有实现任务。

---

## Convention: 不主动运行测试

**What**: 实现功能或修复 Bug 时，不主动编写或运行测试，除非用户在任务描述中明确说明需要测试。

**Why**: 本项目无测试框架配置（无 vitest/jest），运行测试无实际收益且浪费时间。AI 应将精力集中在代码实现的正确性上，而非搭建测试基础设施。

**Example**:

```
# ❌ 错误做法：用户要求实现一个新页面，AI 额外跑了 vitest 写测试
用户: "实现弹药详情页"
AI:   实现功能 → 安装 vitest → 写测试 → 跑测试 → 完成

# ✅ 正确做法：直接实现，不跑测试
用户: "实现弹药详情页"
AI:   实现功能 → 检查 TypeScript 编译无误 → 完成
```

**Exception**: 用户明确说"加个测试"、"跑一下测试"、"确保测试通过"时，才需要执行测试流程。

---

## Convention: 不擅自开启新端口运行项目

**What**: 不要在实现任务中自行启动 `npm run dev` 或 `vite` 等开发服务器，除非用户明确要求运行。

**Why**: 开发服务器通常由用户自己在另一个终端运行（可能已有端口被占用）。AI 擅自开启新端口会导致端口冲突、资源浪费，且用户已有热更新的工作实例，无需重复启动。

**Example**:

```
# ❌ 错误做法：用户要求修复 Bug，AI 先启动 dev server
用户: "修复分类页图片不显示"
AI:   npm run dev (port 5174) → 查看页面 → 修复 → 完成

# ✅ 正确做法：直接修复代码，用户自己验证
用户: "修复分类页图片不显示"
AI:   分析代码 → 修复 → 告知用户改动内容 → 用户在自己的 dev server 上验证
```

**Exception**: 用户明确说"启动项目"、"跑一下看看"、"我需要预览"时，才启动开发服务器，且优先使用端口 `5173`。
