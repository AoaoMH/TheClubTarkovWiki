# Git Commit 信息生成规范（AI 专用指令）

## 核心原则
1. **提交描述语言**：提交的 **标题（Subject）** 和 **正文（Body）** **必须使用中文**进行描述，以增强团队可读性。
2. **规范标准**：遵循 Conventional Commits（Angular 风格）。

## 格式模板（请按此结构生成）
标题行：
`<type>(<scope>): <中文简短标题>`

正文（列表形式）：
- `<中文描述具体变更点 1>`
- `<中文描述具体变更点 2>`
- `<中文描述具体变更点 3>`

## 类型（type）定义
- `feat`: 新功能
- `fix`: 修复 Bug
- `docs`: 文档变更
- `style`: 代码格式（不影响功能）
- `refactor`: 重构（不是新功能也不是修 Bug）
- `perf`: 性能优化
- `test`: 测试相关
- `chore`: 构建/工具/依赖变更

## 范围（scope）定义
- 若涉及数据接口，使用 `API`；涉及界面组件，使用 `UI`，以此类推。
- 如果是针对某个模块比如武器详情可以写为 `WeaponDetail`，以此类推。

## 强制要求（重点）
1. **标题必须以中文动词开头**，例如：“增加”、“修复”、“更新”、“重构”、“移除”。
2. **必须突出中文显示适配**：如果变更增加或修改了任何前端文案、弹窗提示、占位符、槽位名称等，**必须在正文末尾或首行强制说明：“此提交确保所有新增字段均支持中文显示”**（可用加粗或星号强调）。
3. **列表项语言**：正文中的列表项（以 `-` 开头）必须使用中文描述变更内容。

## 示例（严格参照此风格）
**输入变更内容**：
- 新增 fetchItemNames API
- 增加槽位中文/英文语言包
- 修复图片路径损坏

**AI 应生成的提交信息**：
feat(API): 增加装备槽位本地化与物品名称查询功能

- 新增 fetchItemNames 接口，用于加载外部物品名称数据
- 为装备槽位类型补充中文本地化映射及英文回退方案
- 新增槽位冲突类型解析逻辑并生成对应输出
- 清理物品数据，移除失效的图片引用路径
- 添加异步钩子以支持物品名称的动态查询
